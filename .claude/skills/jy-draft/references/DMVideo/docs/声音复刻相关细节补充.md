# 声音复刻与语音合成使用指南

声音复刻与语音合成是紧密关联的两个独立步骤，遵循 **“先创建，后使用”** 的核心流程。

## 1. 准备录音文件
在开始之前，请确保您的音频文件符合 **[声音复刻：输入音频格式]** 的要求（如 WAV/MP3/M4A，时长 10-20 秒，无背景噪音等）。

*   **上传要求**：将符合格式的音频文件上传至公网可访问的位置（例如：阿里云对象存储 OSS）。
*   **访问权限**：必须确保生成的 URL 是 **公开可访问** 的，以便接口能够读取音频数据。

## 2. 创建音色
调用 **创建音色接口** 进行模型训练。

*   **关键参数**：必须指定 `target_model` (或 `targetModel`)。
    *   *作用*：声明创建的音色将由哪个具体的语音合成模型驱动。
*   **前置检查**：若已拥有创建好的音色，可调用 **查询音色列表接口** 确认，若存在则可跳过此步骤。

## 3. 使用音色进行语音合成
当音色创建成功后，系统将返回唯一的标识符 `voice_id` (或 `voiceID`)。

### 3.1 获取标识符
*   **返回值**：`voice_id` / `voiceID`
*   **用途**：该 ID 可直接作为 **语音合成接口** 或各语言 **SDK** 中的 `voice` 参数。

### 3.2 执行合成
使用该 ID 进行文本转语音（TTS），支持多种调用形态：
*   非流式合成
*   单向流式合成
*   双向流式合成

> **⚠️ 重要约束**
> 合成时指定的 **语音合成模型** 必须与 **创建音色时** 指定的 `target_model` **完全一致**。
>
> *   若不一致，合成请求将失败。

> *   **注意**：如果您在其他地域（如海外节点或非北京节点）调用这两个模型，请求将会失败。请在创建音色或合成语音时确保选择了正确的地域配置。

# 声音复刻：输入音频格式

高质量的输入音频是获得优质复刻效果的基础。

| 项目 | 要求 |
|------|------|
| **支持格式** | WAV (16bit), MP3, M4A |
| **音频时长** | 推荐 10~20 秒，最长不得超过 60 秒 |
| **文件大小** | ≤ 10 MB |
| **采样率** | ≥ 16 kHz |
| **声道** | 单声道 / 双声道，双声道音频仅处理首声道，请确保首声道包含有效人声 |
| **内容** | 音频必须包含至少 5 秒连续清晰朗读（无背景音），其余部分仅允许短暂停顿（≤2 秒）；整段音频应避免背景音乐、噪音或其他人声，确保核心朗读内容质量；请使用正常说话音频作为输入，不要上传歌曲或唱歌音频，以确保复刻效果准确和可用。 |
